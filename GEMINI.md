# Project Mandates

## GitHub Synchronization
- All changes must be pushed to the remote repository (GitHub) at the end of each session or after significant updates.
- Use `git push origin main` (or the active branch) to ensure the remote is in sync with the local state.
- Before starting work, always verify sync status with `git pull`.

## Development Workflow
- Follow the phased implementation plan and update `AGENT_PROGRESS_LOG.md` after each session.
- Ensure all tests pass (`npm test`) and linting is clean (`npm run lint`) before pushing.
